## 발견·결정사항

### 결정 1 — `useMyReviewQuery` 는 `retry: false` + 404 빈 상태 분기
- 계약 §6: `/reviews/my` 는 review 없으면 404 (E9804447). 기본 RQ retry(3회)는 404 에 무의미 + 빈 상태 표시 지연 → `retry: false`.
- `errorMapping.isReviewNotFound(err)` 로 404 만 EmptyState("HR/매니저가 평가를 생성하면 시작됩니다"), 그 외 에러는 메시지 표면. (kpi.ts 의 my-assignments 는 빈 배열 반환이라 404 분기 불요였으나, review/my 는 단건 404 라 신규 분기.)

### 결정 2 — 점수 입력 상태(`overrides`)는 매니저 폼 로컬, BE 가 점수 SoT
- 계약 §5: BE 가 유일 계산자. FE per-item managerScore 는 `Record<assignmentId, number|''>` 로컬 상태로 관리.
- 초기화 = `useReviewKpiItemsQuery` 의 저장 managerScore (`item.managerScore ?? ''`). `useEffect([review.id, managerComment, items])` 동기화.
- `previewKpiScore(items, overrides)` = §5 산식 그대로(`itemScore = managerScore ?? autoScore`, `Σ(itemScore×weight)/Σ(weight)`, 비-NULL 만) — **표시 전용** Card 프리뷰. 제출(submit-manager)·임시저장(PATCH itemScores)에 동일 입력값 전달, 최종 kpiScore 는 BE 응답이 SoT (previewHint i18n 명시).
- managerScore NumberInput placeholder = autoScore → "비우면 autoScore 폴백" UX 가시화.

### 결정 3 — 편집 가능 상태 게이트 (계약 §3/§6)
- 자기평가: `editable = status === 'SELF_PENDING'` 일 때만 selfComment 편집·임시저장·제출 노출. 그 외 = readOnly Textarea + 잠금 Alert.
- 매니저 채점: `editable = status === 'MANAGER_PENDING'` 일 때만 score-input 표 + managerComment 편집 + 저장·제출. 그 외 = readOnly 표 + notEditable Alert (비교 탭은 항상 표시).
- transition 메뉴는 §3 매트릭스(`REVIEW_TRANSITIONS` 4개)만 — submit-self/submit-manager 는 전용 엔드포인트라 메뉴 미노출(전용 버튼).

### 결정 4 — 매니저 목록 행 클릭 선택 + Tabs 패널 (Wireframe 2 정합)
- 비교 view 는 별도 라우트 없음 — 매니저 폼 내 `<Tabs>` 2탭(KPI 채점 / 자기↔매니저 비교)으로 구현(계약 §7 명시).
- 행 클릭 = `onClick` row + 선택 행 배경 강조. Actions 셀(transition 메뉴)은 `onClick stopPropagation` 으로 행 선택과 분리.
- 패널은 `key={selectedReview.id}` 로 review 전환 시 폼 상태 리셋(stale override 방지).

### 결정 5 — 생성 모달 개별/일괄 단일 컴포넌트 (SegmentedControl)
- 계약 §6: 개별(`{employeeId}`) + 일괄(`{employeeIds[]}`, skip 카운트 반환). SegmentedControl 1 모달 2 모드.
- 일괄 입력 = Textarea + `split(/[\s,]+/)` 파싱(줄바꿈·콤마·공백). bulk 응답 `{createdCount, skippedCount}` → 알림 "생성 N건 / 건너뜀 M건". skip 은 에러 아님(계약 명시).

### 결정 6 — `@mantine/modals` 미설치 → core Modal 인라인 confirm (P0-S1/S2 정합)
- 제출 확인(자기평가) = `@mantine/core::Modal` 인라인 confirm state. dependency 추가 0.
- Mantine v9: Grid `gutter` 미사용(SimpleGrid spacing / Grid Col span 객체), Tabs/Textarea autosize/LoadingOverlay 표준 API.

### 잠재 후속
- BE 미구현(병렬) 상태 — FE 호출 시 404/네트워크. BE P0-S3 완성 후 통합 스모크.
- employeeId 입력은 P0-S6 principal 주입 전 임시(자기평가 employeeId 입력 / transition actorEmployeeId 미전달 = BE default). principal 도입 시 자기평가는 입력 제거 + 매니저 transition 의 actorEmployeeId 자동 주입.
- `kpi-items` 동결 우선(계약 §5): FINALIZED/MANAGER_SUBMITTED 이후 BE 가 저장 스냅샷 반환 → FE 는 동일 컴포넌트로 렌더(별도 분기 불요, 저장값이 그대로 표시됨).
- APPEAL_*/ARCHIVED 상태는 Badge·라벨만 박제(P0-S10/P2-S3 전이). transition 메뉴에는 미노출(§3 4개 한정).
- openapi-typescript codegen 도입 시 본 슬라이스 수동 타입 → generated 마이그레이션.
