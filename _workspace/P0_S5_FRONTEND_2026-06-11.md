## 발견·결정사항

### 결정 1 — content 는 BE 동결 스냅샷 표시만 (FE 재계산 0)
- 계약 §5: `content` jsonb 는 BE 가 발행 시점에 review 확정값 + 전사 분포를 동결한 스냅샷. FE 는 **표시만** — distribution 비율(0~1) → % 환산 렌더(`formatReportRatio`)와 score `toFixed`(reviews.ts `formatScore` 재사용)만 허용.
- finalGrade/finalScore/kpiScore/managerComment/kpiItems/distribution 전부 `content` 안에서 직접 렌더. review 라이브 값(reviews API) 미참조 — 발행 후 review 변경에 리포트 불변(append-only/supersede 만 정정).
- mbo/competency/mra 는 P0 = null → "추후 제공"(`card.scoreP1`) 안내 타일. finalScore 미산출 시 `formatScore` 의 '—'.

### 결정 2 — cycle FINALIZED 게이트는 FE 선차단 (BE E9804252 이중 차단)
- 계약 §3: publish/supersede 는 `cycle.status == FINALIZED`. FE 는 `useCyclesQuery` 의 `content` 에서 선택 cycle status 를 찾아 `cycleFinalized` 판정.
- 일괄 발행 버튼·행별 재발행 버튼은 `!cycleFinalized` 시 `disabled` + 안내 Alert(`needFinalized`). 발행 모달은 FINALIZED 아니면 경고 Alert + 제출 버튼 비활성.
- 위반은 BE E9804252(422)로도 차단 — FE 선차단은 UX 명확화. cycle status 라벨은 기존 `cycles.status[...]` i18n 재사용.

### 결정 3 — `ReportDistributionBars` 신규 (기존 DistributionBars 재사용 불가 — 키 단위 상이)
- 계약 §7(#7): content.distribution = **비율(0~1)** 단일 + 건수 비노출(E9 소규모 역추적 방지) + UNRATED 없음(FINALIZED 는 grade 보유) + 본인 등급 강조.
- 기존 `calibration/DistributionBars` 는 `current`=건수(int) + `target`=비율 듀얼 표기 구조(P0-S4) — distribution 이 비율 단일이고 본인 강조가 필요한 본 슬라이스에 그대로 적용 불가 → 비율 전용 경량 컴포넌트 신설(Progress 수동, `@mantine/charts` 미설치 정합 — calibration 과 동일 막대 관행).
- GradeBadge·CycleSelect·ReviewKpiItemsTable 은 **무변형 재사용**. DistributionBars 만 변형 신설(계약 "변형 필요 시 박제" 조항 충족).

### 결정 4 — 자동 view 1회 = useEffect + ref 가드 (무한 재호출 금지)
- 계약 §7 주의: view 자동 호출은 `viewedAt null && 최초 로드 시 1회`. `useRef<string | null>` 로 view 발행한 report.id 추적 — `report.id` 가 같고 이미 발행했으면 skip.
- deps = `[report?.id, report?.viewedAt]`. view mutation 성공 → invalidate → 재조회 시 viewedAt 채워짐 → `report.viewedAt != null` 가드로 재호출 차단(ref 와 이중 안전). `viewMut` 은 비안정 참조라 deps 제외(eslint-disable 주석 명시).
- 컴포넌트 `key={cycleId:employeeId}` 로 조회 대상 변경 시 ref 리셋(언마운트→재마운트) — 다른 사원 조회 시 자동 view 재발동 정상.

### 결정 5 — acknowledge 멱등 + 완료 후 비활성
- 계약 §6: acknowledge 는 `acknowledged=true + acknowledged_at`(멱등). FE 는 `report.acknowledged` 시 버튼 `disabled` + 라벨 "확인 완료" + Badge + `acknowledgedAt` 시각 표시.
- mutation onSuccess → invalidate(detail + all) → my 재조회로 acknowledged 반영. superseded 행 acknowledge 시도는 BE E9804934(409) → notification(본 화면 my 는 active 행만 받으므로 정상 경로 미발생, 동시성 방어).

### 결정 6 — 발행 요약 = FINALIZED review 수(reviews 재사용 클라 필터) vs active 발행 수
- 계약 §7(#26): 요약 = useReviewsByCycleQuery 재사용 클라이언트 필터. `reviews.filter(status === 'FINALIZED').length` = 발행 대상 수, `reports.filter(!superseded).length` = active 발행 완료 수.
- reports 목록은 superseded 포함 전부 반환(§6 — HR 이력 가시성, publishedAt DESC). 테이블은 전부 표시 + superseded Badge, 요약·재발행 버튼은 active 기준.

### 잠재 후속
- BE 미구현(병렬) 상태 — FE 호출 시 404/네트워크. BE P0-S5 완성 후 통합 스모크.
- `actorEmployeeId` 는 P0-S6 principal 주입 전 전부 null(BE default). publish/acknowledge/supersede 의 actorEmployeeId 는 principal 도입 시 자동 주입으로 대체.
- my report 의 `employeeId` 는 P0-S6 principal 수신 전 plain UUID 입력 — principal 도입 후 본인 고정(입력 제거). HR 현황 테이블 employeeId 라벨 join 도 rm_employee 수신 후.
- 부서 분포는 P0-S6 rm_org 수신 전 → 전사 분포 % 로 대체 박제(§0-G_PERF_E9). org 단위 분포는 후속.
- openapi-typescript codegen 도입 시 본 슬라이스 수동 타입 → generated 마이그레이션.
