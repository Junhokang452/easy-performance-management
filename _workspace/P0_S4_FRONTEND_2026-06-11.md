## 발견·결정사항

### 결정 1 — `effectiveGradeLabel` 은 본부 grid 행 표시 보조 (BE 분포 계산은 불변 SoT)
- 계약 §7(#19): review 행에 `effectiveGrade Badge` 표시. 단 reviews 목록(`useReviewsByCycleQuery` 재사용)은 `finalGrade`·`kpiScore` 만 반환하고 effectiveGrade 를 직접 주지 않음.
- §5 effectiveGrade 정의 = `finalGrade ?? bandGrade(kpiScore)` (밴드 S≥90/A≥80/B≥70/C≥60/D<60, null→UNRATED) 그대로 `effectiveGradeLabel(finalGrade, kpiScore)` 표시 헬퍼로 구현.
- **분포·시뮬레이션 등급은 BE 가 SoT 불변** — 본 헬퍼는 reviews 목록의 행 라벨 렌더만 보조(adjust/apply 결과는 BE 가 `finalGrade` 에 반영하므로 그 경우 finalGrade 우선). DistributionBars 의 현재/목표/proposed 분포·등급은 전부 BE 응답 그대로 렌더(§7 재계산 금지 준수 — 비율 환산만 % 표시용).

### 결정 2 — transition 메뉴는 §3 2개 한정, CONFIRMED 진입은 confirm 모달 / ADJUSTED 는 자동 승격
- `CALIBRATION_TRANSITIONS` = `{ PLANNED: [IN_SESSION], CONFIRMED: [CLOSED] }` 만. IN_SESSION/ADJUSTED→CONFIRMED 는 confirm endpoint 전용이라 transition 메뉴 비노출(시도 시 BE E9804246).
- IN_SESSION→ADJUSTED 는 adjustments 첫 호출 시 BE 자동 승격 — 메뉴·버튼 미노출. confirm 버튼은 `canConfirmSession(status)` (IN_SESSION/ADJUSTED) 일 때만 표면.

### 결정 3 — 조정 가능 게이트 3중 (세션 상태 + review 상태 + 세션 선택)
- 계약 §3/§5: adjustments 는 CALIBRATION 단계 + 세션 IN_SESSION/ADJUSTED + 대상 review.status==CALIBRATION.
- FE 행별 AdjustGradeMenu `enabled` = `세션 IN_SESSION|ADJUSTED && sessionId != null && review.status === 'CALIBRATION'`. 위반 시 메뉴 비활성(버튼 disabled). 단계 위반은 BE E9804250/E9804933 으로도 이중 차단.

### 결정 4 — `@mantine/charts` 미설치 → DistributionBars 수동 가로 막대
- 계약 §7 명시: Progress/Box 수동 가로 막대. `Progress.Root`+`Progress.Section` 으로 버킷별 현재 비율(건수/총합) 렌더 + 목표 비율은 별도 텍스트 마커 병기(겹침 막대 대신 듀얼 표기 — 가독성).
- 분포 객체 키 단위 정합: current/resulting/actual = 건수(int) → 막대 % 는 `count/total` 환산(렌더 전용, §7 허용 범위). target = 비율(0~1) → `formatRatio` 로 % 표시. UNRATED 는 회색 + 목표 비율 대상 제외.

### 결정 5 — SessionFormModal 단일 컴포넌트 생성/수정 (ReviewCreateModal 패턴)
- 계약 §6: 생성·수정 동일 body(`ownerOrgUnitId?/scheduledAt?/participantIds?`). `session` prop 유무로 isEdit 분기 — create/update mutation 양쪽 보유, 제목·노트·버튼 라벨만 분기.
- participantIds = `TagsInput`(@mantine/core 존재 확인 후 사용 — 줄단위 Textarea 폴백 불요). scheduledAt = `DateTimePicker`(@mantine/dates) → `Date` 상태 → `.toISOString()` timestamptz 송신. 빈 값은 null 송신(생성·수정 동형).

### 결정 6 — forcedSupported 게이트로 simulate/apply 사전 비활성
- 계약 §4: ABSOLUTE 모드(E9804248) 또는 비-S_A_B_C_D 척도(E9804249) 는 강제 배분 불가.
- distribution 응답의 `distributionMode != 'ABSOLUTE' && ratingScale === 'S_A_B_C_D'` 일 때만 simulate/apply 버튼 활성 + 그 외 안내 Alert. (BE 가 거부하지만 FE 선차단으로 UX 명확화 — 등급 계산 자체는 BE SoT 불변.)

### 잠재 후속
- BE 미구현(병렬) 상태 — FE 호출 시 404/네트워크. BE P0-S4 완성 후 통합 스모크.
