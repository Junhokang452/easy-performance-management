## 발견·결정사항

### 결정 1 — enum 모듈 분리 (`kpiEnums.ts`)
- `kpi.ts` 가 페이지·컴포넌트에서 enum + 타입을 빈번히 import + re-export 하므로 enum/상수만 별도 `kpiEnums.ts` 로 분리.
- `kpi.ts` 는 이를 `export type {...} from './kpiEnums'` + `export {ALL_*} from './kpiEnums'` 로 재노출 → 소비자는 `from '../api/kpi'` 단일 import 유지(표면 단순화).

### 결정 2 — flat nodes 클라이언트 조립 (`buildTreeNodeViews`)
- 계약 §4: `KpiTreeDetailResponse.nodes` 는 flat 배열. FE 가 `parentId` 로 부모-자식 순서 평탄화 + depth 부여.
- 정렬 = 형제 간 `label` 오름차순 안정 정렬. **고아 노드 안전장치**: 부모가 트리 내 부재한 노드는 root(depth 0)로 폴백 추가(누락 방지).
- 들여쓰기 = depth × 24px + depth>0 좌측 border (시각적 계층).

### 결정 3 — per-parent 가중치 합 뱃지 = BE 파생값 표시만
- 계약 §3/§4: `childWeightSum`/`childWeightComplete` 는 **BE 응답에 포함** → FE 재계산 금지(표시만).
- 뱃지 배치 = **자식 보유 노드(부모)에만** (`hasChildren` Set). complete(==1.0)=green filled / exceeded(>1.0001)=red filled / incomplete(<1.0)=yellow light.
- `effectiveWeight`·`achievementRate`·`latestActualValue` 도 BE 계산값 그대로 렌더(MyKpiAssignmentResponse).

### 결정 4 — 실적 정정 = supersede 전용 (append-only)
- 계약 §4: actuals UPDATE/DELETE 없음. 정정 = `POST /kpi-actuals/{id}/supersede` (신규 row + supersedesId).
- `ActualFormModal` 단일 컴포넌트 2 모드: `report`(create) / `correct`(supersede, 원본 prefill).
- `ActualHistoryModal` 에서 `superseded=true` 행은 정정 버튼 비노출(E9804925 사전 가드) + '정정됨' 뱃지 / 유효 행은 '최신' 뱃지 + '정정' 버튼.

### 결정 5 — `@mantine/modals` 미설치 → core Modal 인라인 confirm (P0-S1 정합)
- 삭제 확인(트리/노드/배정) 전부 `@mantine/core::Modal` 인라인 confirm state 패턴. dependency 추가 0.

### 결정 6 — source = MANUAL 고정 (P0)
- 계약 §3: P0 는 node source MANUAL 외 거부(E9804239). `NodeFormModal` 생성 시 `source: 'MANUAL'` 하드코딩(선택 UI 비노출) — HCM/EXTERNAL 셀렉트는 P1.
- `ALL_KPI_NODE_SOURCES` 상수는 박제(향후 P1 노출용)하되 폼에서 미사용.

### 결정 7 — My KPI 입력/조회 분리 (SelfEvaluation employeeId 패턴)
- cycleId Select + employeeId TextInput 은 **입력 상태**, '조회' 버튼 클릭 시 `applied` 로 확정 → `useMyKpiAssignmentsQuery(applied.cycleId, applied.employeeId)` (둘 다 필수, principal 주입 P0-S6 이후).
- 미입력 시 안내 텍스트, enabled 가드로 빈 쿼리 차단.

### 결정 8 — 본부 BSC 그룹핑 = 관점별 묶음 (계층 해체)
- 계약 §5: BSC ON = bscPerspective 4 관점 그룹핑(+미지정). 관점별 그룹은 계층(parent-child)을 해체하므로 **SimpleGrid 4 컬럼 + 관점별 노드 평면 리스트**(label 정렬, weight 뱃지)로 렌더.
- BSC OFF = KpiNodeTree readOnly (계층 트리). 토글 기본값 = `tree.bscEnabled`(useEffect 동기화).

### 잠재 후속
- BE 미구현(병렬) 상태 — FE 호출 시 404/네트워크. BE P0-S2 완성 후 통합 스모크.
- 노드 weight 수정 시 형제 합 초과(E9804238)는 BE 가드 → FE 는 ApiError 메시지 표면(사전 클라 합산 검증은 미도입, 트리 점진 구축 허용 정책 정합).
- assignment weight/target override 수정(PATCH)은 본 슬라이스에서 추가 폼 미노출(생성+삭제만 UI 제공) — 수정 mutation(`useUpdateAssignmentMutation`)은 박제만. P1 인라인 편집 후속.
- 드래그·드롭 cascade 재배치(#13) P1 보류(계약 명시).
- openapi-typescript codegen 도입 시 본 슬라이스 수동 타입 → generated 마이그레이션.
