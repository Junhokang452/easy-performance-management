## 발견 / 결정사항

### D1 — `ReviewService.bandGrade(BigDecimal)` public 신설 + 재사용 (P0-S1 D1 함정 회피)
계약 §5: effectiveGrade = `finalGrade ?? bandGrade(kpiScore)` 의 밴드 = "ReviewService 의 밴드 로직 public
재사용 (중복 구현 금지)". **결정**: 기존 `computeFinalGrade(tenantId, cycleId, score)` 는 policy.ratingScale
게이트(S_A_B_C_D 일 때만 등급) 가 묶여 있어 그대로 재사용 불가 — calibration 의 effectiveGrade 는 policy
무관 순수 점수→밴드 환산이 필요. 따라서 **순수 산식 `public String bandGrade(BigDecimal score)`**(S≥90/A≥80/
B≥70/C≥60/D<60, null→null) 를 신설하고 `computeFinalGrade` 가 policy 게이트 후 이 메서드에 위임하도록 리팩터.
`CalibrationService` 는 `ReviewService` 생성자 주입 → `reviewService.bandGrade(kpiScore)` public 호출만
(cross-package private 0). effectiveGrade 에서 band 가 null(kpiScore null) 이면 `"UNRATED"` 버킷.

### D2 — jsonb 4컬럼 (P0-S1 D2 패턴) + 전용 `CalibrationJson` 헬퍼 분리
4 jsonb 컬럼: `participant_ids`(UUID[]) / `adjustment_log`(append) / `policy_distribution`+
`actual_distribution`+`simulation_log`(분포·이력). 전부 `@JdbcTypeCode(JSON) + String` 매핑 + service
ObjectMapper `USE_BIG_DECIMAL_FOR_FLOATS`(비율 정밀도 보존). ReviewService 가 1개 jsonb(kpi_score_detail)
를 service 내부에서 처리한 것과 달리 본 슬라이스는 4컬럼 × 6종 변환(serialize/parse × 비율/카운트/배열)이라
**`CalibrationJson` 으로 분리**(서비스 가독성). entry 의 `at` 시각 직렬화에 `JavaTimeModule` +
`WRITE_DATES_AS_TIMESTAMPS` disable(ISO-8601). 손상 jsonb 행은 graceful null/빈 리스트 폴백. 분포 맵은
parse 시에도 정준 순서 5등급 채움(FE 막대 렌더 정합).

### D3 — largest remainder method 별도 클래스 `DistributionMath` (Σquota==N SoT)
계약 §5 강제 배분 = floor(N×ratio) 우선 + 잔여 소수부 큰 순(동률 정준 S→A→B→C→D) +1. **Σquota == N 을
수학적으로 보장**하는 것이 핵심 불변식이라 순수 함수로 분리하고 **경계 케이스를 전수 테스트**(N=0 → Σ=0 /
N=1 → 단일 최대잔여 배분 / floor 합 < N → 잔여 분배 / N ∈ {2,5,7,13,100,999} → Σ==N 항상). target 검증
(§4 INVALID_TARGET): 키 ∉ {S,A,B,C,D} / 합 ≠ 1.0±0.001 / 음수 / null·empty + 통과 시 5등급 정규화. BigDecimal
floor 는 `setScale(0, FLOOR)` + `intValueExact()`.

### D4 — apply 잠금·재적용 정책 (계약 §0-2) + simulate 무저장 (§0-1)
- **simulate** = 무저장 순수 계산 (`@Transactional(readOnly=true)`, `distributionRepository.save` 호출 0 —
  테스트 `never()` 가드). proposed 목록 + 결과 분포만 반환.
- **apply** = 대상 review.finalGrade **일괄 UPDATE (status 불변** — CALIBRATION 유지, FINALIZED 전이 아님)
  + RatingDistribution **upsert**(전사 NULL-org 행 — `findByTenantIdAndCycleIdAndOrgUnitIdIsNull` →
  없으면 신규) + `simulation_log` **append**(멱등 아님 — 재적용마다 entry 보존, 테스트로 2 entry 검증).
- **게이트** = simulate/apply 모두 `cycle.status == CALIBRATION` 한정(§3, `requireCycleStage` 가 policy
  검증보다 먼저 발동) + `policy.distributionMode != ABSOLUTE`(MODE_NOT_FORCED) + `ratingScale == S_A_B_C_D`
  (SCALE_NOT_SUPPORTED). skippedCount = cycle 의 CALIBRATION 상태 중 kpiScore NULL 수.

### D5 — 상태기계 자동 승격 + confirm 전용 전이 (계약 §3)
- **IN_SESSION → ADJUSTED** = adjustments API 첫 호출 시 서비스 자동 승격(transition 으로 진입 불가).
  adjust 는 review.finalGrade=toGrade + adjustment_log append(fromGrade = 조정 전 effectiveGrade — D1
  bandGrade 로 산출). 대상 review 는 CALIBRATION 상태여야(아니면 CALIBRATION_REVIEW_NOT_READY 409) + 세션
  cycle 소속이어야(아니면 CALIBRATION_ADJUSTMENT_INVALID).
- **IN_SESSION/ADJUSTED → CONFIRMED** = confirm endpoint 전용 (transition 으로 시도 시
  CALIBRATION_INVALID_STATUS_TRANSITION). `finalizeReviews==true` 면 cycle 의 CALIBRATION review 전부에
  **P0-S3 `reviewService.transition(reviewId, FINALIZED)` 행 단위 재사용**(public 경로 — cross-package
  private 금지, kpiScore NULL 행 skip 카운트). CONFIRMED/CLOSED 세션은 adjust/PATCH/confirm 재시도 시
  CALIBRATION_SESSION_LOCKED 409.

### D6 — Clock 주입 2-생성자 (P0-S3 ReviewService 패턴 정합)
`CalibrationService` 2-생성자 (운영 `Clock.systemUTC()` + 테스트 고정 Clock) → `CalibrationJson(clock)` 으로
전파. adjustment/simulation entry 의 `at` + confirmedAt/appliedAt 테스트 친화 고정(FIXED

### D7 — `ReviewService` FINALIZED 전이 보존 보강 (계약 §5 명시 변경)
`finalizeReview` 가 `finalGrade = (기존 != null ? 유지 : bandGrade via computeFinalGrade)` 로 변경 — 캘리브
조정(adjust)·강제 배분(apply)이 review.finalGrade 를 미리 세팅한 뒤 confirm finalize 로 FINALIZED 전이될 때
