## 핵심 결정 / 발견

### D1 — tenant 해석은 performance `com.easyperformance.common.TenantSupport` 사용 (talent 동등물 실재)
performance 에 이미 talent `TenantSupport` 와 동일 시그니처의 `com.easyperformance.common.TenantSupport.currentTenantId()` 존재 (단계 1 도입, `FALLBACK_TENANT_ID = 00000000-...-0001`). 기존 도메인 service 전부가 이를 통해 tenantId 획득. 신규 클래스 도입 없이 그대로 사용. S2S 요청에는 JWT 없음 → `TenantContext` null → fallback 테넌트(단일 DB 가정). 단계 2 Model B 진입 시 fallback 제거는 별개 슬라이스(TenantSupport 공통 정합).

### D2 — HMAC 은 raw body 기준 (@RequestBody String 수신 → 인증 후 파싱)
talent 그대로: 컨트롤러가 `@RequestBody String rawBody` 로 받아 `SyncChannel.authenticate` 가 raw 문자열에 HMAC 검증 → 통과 후에만 `ObjectMapper.readValue` 역직렬화. 객체 바인딩 후 재직렬화 검증 금지(서명 불일치 위험) — 계약 §주의 준수. lib `HmacService(secret).verify(rawBody, signature)` 위임(import `com.easyware.platform.hmac.HmacService`, talent 동일).

### D3 — rm_* id 수동 set = 소스 SoR id (Spring Data isNew 함정 — talent 가 해소한 방식 그대로)
rm entity 의 `@Id` 는 UuidV7 생성이 아닌 외부 주입(hcm SoR id 를 `setId`). talent 는 `findById(id).orElse(null)` 분기로 명시 insert/update 처리 — `save()` 만 호출하되 부재 시 신규 객체를 `new` 하므로 merge 경로 OptimisticLocking 우려 없음(`TenantAwareAuditEntity` 는 `@Version` 부재 — talent S15 실측). recruit `e986533` ④ 의 @PrePersist 우회는 불필요(talent 가 이미 분기 조회로 처리). 본 슬라이스도 동일 사본.

### D4 — SecurityConfig 최소 변경 + JwtAuthFilter `shouldNotFilter` 불요
performance `JwtAuthFilter` 는 `shouldNotFilter` 가 없으나, 토큰 부재 시 인증 미설정으로 통과(예외 없음)하므로 `/api/internal/**` 무영향. 필요한 변경은 SecurityConfig 의 `/api/internal/**` permitAll 1줄 뿐(talent 와 동일 매처 위치). 자체 3중 가드가 인증을 담당.

### D5 — application.yml 만 수정 (prod yml s2s 관례 정합)
기존 `performance.s2s.easytalent.*`(송신, P0-S6 이전) 가 `application.yml` 에만 있고 `application-prod.yml` 엔 없는 관례 그대로 → 신규 `performance.s2s.hcm.*`(수신) 도 `application.yml` 에만 추가. prod 는 env(`HCM_S2S_BEARER_TOKEN`/`HCM_S2S_HMAC_SECRET`) 로 주입(미설정 시 빈 → 503 차단). `performance.s2s` 키 아래 `easytalent`(송신) ↔ `hcm`(수신) 형제 배치.

### D6 — 스캔 무변경 (App 변경 0)
`PerformanceManagementApplication` 의 `scanBasePackages`/`@EntityScan`/`@EnableJpaRepositories` 가 모두 `com.easyperformance` 를 포함 → 신규 `com.easyperformance.{sync,readmodel}.*` 패키지 자동 등록. App 클래스 변경 불요(easy-job `1ddf97e` scanBasePackages 함정과 무관 — 별도 @ComponentScan 미사용).
